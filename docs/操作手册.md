# 野火客服工作台 — 部署与操作手册

## 1. 系统架构

```
用户 App (野火 IM)
    │
    │ ① 给"客服机器人"发消息
    ▼
┌────────────────────────────────────────────┐
│          Docker                             │
│  ┌──────────────┐   ┌──────────────┐       │
│  │  im-server   │   │  app-server  │       │
│  │  :80 :1883   │   │  :8888       │       │
│  │  :18080      │   │  登录/注册    │       │
│  └──────┬───────┘   └──────────────┘       │
│         │                                   │
│         │ ② webhook POST 回调               │
└─────────┼───────────────────────────────────┘
          │
          ▼
┌────────────────────────────────────────────┐
│  客服工作台 (Next.js :3000)  ← 本地运行     │
│                                             │
│  /api/webhook/robot   ← 接收用户消息        │
│  /api/sessions        ← 会话管理            │
│  /api/sse             ← 实时推送给客服       │
│                                             │
│  ③ 客服回复 → Robot API → im-server → 用户  │
└────────────────────────────────────────────┘
```

## 2. 环境要求

| 工具 | 版本 | 用途 |
|------|------|------|
| Docker / Docker Compose | 最新 | 运行 IM Server + App Server |
| Node.js | 18+ | 运行客服工作台 |
| PostgreSQL | 14+ | 客服工作台数据库 |

## 3. 部署步骤

### 3.1 启动野火 IM Server

```bash
cd deploy
docker compose up -d
```

等待 10 秒左右，验证服务：

```bash
# 确认容器运行中
docker compose ps

# im-server 日志应显示 "Wildfire IM server start success"
docker logs wfc-im-server --tail 5

# app-server 日志应显示 "Started Application"
docker logs wfc-app-server --tail 5
```

### 3.2 配置 im-server

首次启动后需要修改 `server.ip` 为你的局域网 IP，否则手机 App 连不上：

```bash
# 查看你的 IP
# macOS:
ipconfig getifaddr en0
# Linux:
hostname -I | awk '{print $1}'

# 修改 server.ip（把 YOUR_IP 替换为实际 IP）
docker exec wfc-im-server sed -i 's/server.ip 0.0.0.0/server.ip YOUR_IP/' /opt/im-server/config/wildfirechat.conf

# 开启机器人回调携带发送者信息
docker exec wfc-im-server sed -i 's/robot.callback_with_sender_info false/robot.callback_with_sender_info true/' /opt/im-server/config/wildfirechat.conf

# 开启拉取远程历史消息
docker exec wfc-im-server sed -i 's/message.remote_history_message 0/message.remote_history_message 1/' /opt/im-server/config/wildfirechat.conf

# 重启生效
docker restart wfc-im-server
```

### 3.3 配置 app-server

app-server 需要连接 im-server 的容器内部地址：

```bash
docker exec wfc-app-server sed -i 's|http://localhost:18080|http://im-server:18080|' /opt/app-server/config/im.properties
docker restart wfc-app-server
```

### 3.4 创建 PostgreSQL 数据库

```bash
psql -U postgres -c "CREATE DATABASE \"wildfire-cs\";"
```

### 3.5 启动客服工作台

```bash
# 项目根目录
# 1. 安装依赖
npm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 修改数据库连接和 IM Server 地址

# 3. 初始化数据库
npx prisma migrate dev

# 4. 创建默认管理员
npm run db:seed

# 5. 启动开发服务器
npm run dev
```

打开 http://localhost:3000，使用 `admin / admin123` 登录。

### 3.6 .env 配置说明

```bash
# 数据库
DATABASE_URL="postgresql://postgres:123456@localhost:5432/wildfire-cs"

# JWT 密钥（生产环境请修改）
JWT_SECRET="wildfire-cs-jwt-secret-change-in-production"

# 野火 IM Server 地址（从宿主机访问 Docker）
WILDFIRE_IM_URL="http://localhost:80"
WILDFIRE_ADMIN_URL="http://localhost:18080"
WILDFIRE_ADMIN_SECRET="123456"

# Webhook 回调地址（IM Server 从 Docker 内部访问宿主机）
# Docker Desktop (Mac/Windows) 用 host.docker.internal
# Linux 用宿主机 IP
WEBHOOK_CALLBACK_BASE="http://host.docker.internal:3000"
```

## 4. 配置客服机器人

### 4.1 在工作台创建机器人

1. 登录 http://localhost:3000（admin / admin123）
2. 点击侧边栏 **机器人图标** → 机器人管理
3. 填写名称和显示名称，点击「创建」
4. Webhook 回调地址会**自动配置**

### 4.2 让用户找到机器人

App 用户无法直接搜索到机器人，需要机器人主动发消息：

1. 机器人管理页面，点击对应机器人的「发送欢迎语」
2. 从下拉列表选择目标用户
3. 点击发送 → 用户 App 聊天列表中出现该机器人
4. 用户点进去即可开始咨询

## 5. 手机 App 配置

### 5.1 下载 App

使用官方 Android/iOS 客户端，编译时修改 IM Server 地址为你的 IP。

### 5.2 登录

App 通过 app-server（端口 8888）进行登录注册：

- 默认短信超级验证码：`66666`
- 输入手机号 → 验证码填 `66666` → 登录成功

## 6. 完整消息流程

```
用户在 App 给机器人发消息
        ↓
  im-server 收到消息
        ↓
  POST http://host.docker.internal:3000/api/webhook/robot
        ↓
  客服工作台收到，创建/更新会话，SSE 通知在线客服
        ↓
  客服在工作台点「接入」→ 输入回复 → 点「发送」
        ↓
  工作台调用 Robot API (im-server:80) 通过机器人发给用户
        ↓
  用户 App 收到回复
```

## 7. 测试（不需要手机 App）

客服工作台内置了测试页面：

1. 登录后点击侧边栏 **扳手图标** → 测试页面
2. 左侧是「客户端模拟器」，右侧是「客服工作台」
3. 设置客户名称 → 发消息 → 右侧出现新会话
4. 右侧点「接入」→ 回复消息 → 左侧收到回复

> 注意：测试页面的客户端模拟器不经过 IM Server，直接写本地数据库。
> 客服回复时会真实调用 Robot API，IM Server 不在线时控制台会报错但不影响本地消息流转。

## 8. 端口清单

| 端口 | 服务 | 谁访问 |
|------|------|--------|
| 80 | im-server HTTP | 手机 App + 客服工作台后端 (Robot API) |
| 1883 | im-server MQTT | 手机 App |
| 8083 | im-server WebSocket | Web 客户端 |
| 8888 | app-server | 手机 App（登录注册） |
| 18080 | im-server Admin API | 客服工作台后端 |
| 3000 | 客服工作台 (Next.js) | 客服人员浏览器 |
| 5432 | PostgreSQL | 客服工作台后端 |

## 9. 常见问题

### App 发消息后工作台收不到

1. **检查 callback 地址**：机器人的 webhook 必须是 `http://host.docker.internal:3000/api/webhook/robot`（不能是 `localhost`，Docker 容器内 localhost 指向容器自己）

2. **检查 server.ip**：`wildfirechat.conf` 中 `server.ip` 必须是你的局域网 IP，不能是 `0.0.0.0`

3. **手动验证 callback**：
   ```bash
   # 从 Docker 内部测试
   docker exec wfc-im-server wget -q -O- http://host.docker.internal:3000/api/webhook/robot --post-data='{}' --header='Content-Type: application/json'
   ```

### App 搜不到机器人

正常现象。机器人需要主动给用户发消息才会出现在用户聊天列表。在工作台机器人管理页使用「发送欢迎语」功能。

### IP 变了怎么办

```bash
# 1. 更新 im-server
docker exec wfc-im-server sed -i 's/server.ip .*/server.ip NEW_IP/' /opt/im-server/config/wildfirechat.conf
docker restart wfc-im-server

# 2. 更新 .env 中的 WEBHOOK_CALLBACK_BASE（如果不用 host.docker.internal）
# 3. 重新设置所有机器人的 callback（在机器人管理页面删除重建）
```

## 10. 停止服务

```bash
# 停止 IM Server
cd deploy && docker compose down

# 停止客服工作台
# 在终端按 Ctrl+C
```

## 11. 生产环境注意事项

- 修改 `http.admin.secret_key`（默认 `123456`）
- 修改 `token.key`（默认 `testim`）
- 关闭超级验证码 `sms.super_code`
- 修改 `JWT_SECRET`
- im-server 切换 MySQL 数据库（`embed.db 0`）
- 启用 HTTPS
